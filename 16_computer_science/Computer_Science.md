# 16. Computer Science

## Degree Programs

| Level | Programs |
|---|---|
| Undergraduate | Computer Science (BS/BA), Computer Engineering, Software Engineering |
| Masters | MS in Computer Science, MS in Software Engineering, MEng in CS, MSCS (professional), MS in Data Science (some CS depts), MS in Artificial Intelligence |
| PhD | Computer Science (with specialization) |

**Note:** Computer Science is the study of computation, information, and automation. It spans theory (what can be computed) to practice (building real systems). One of the highest-demand, highest-salary fields across all of academia and industry.

## 16.1 Core Curriculum (Undergraduate)

### Foundation
Calculus I-II, Linear Algebra, Discrete Mathematics (logic, sets, combinatorics, graph theory), Probability & Statistics.

### Core CS Courses

| Course | Content |
|---|---|
| **Programming (Intro Sequence)** | Programming foundations (variables, control flow, functions, OOP), data structures & algorithms lab, typically in Python/Java/C++ |
| **Data Structures** | Arrays, linked lists, stacks, queues, hash tables, trees (BST, AVL, B-trees), heaps, graphs, implementation & analysis |
| **Algorithms** | Sorting, searching, graph algorithms (BFS, DFS, Dijkstra, MST), dynamic programming, greedy algorithms, divide & conquer, NP-completeness, approximation algorithms |
| **Computer Architecture** | Digital logic, combinational/sequential circuits, ISA design, pipelining, caches, memory hierarchy, multiprocessors, RISC-V |
| **Operating Systems** | Processes/threads, scheduling, synchronization (locks, semaphores, monitors), memory management (virtual memory, paging), file systems, I/O |
| **Discrete Mathematics & Theory** | Formal languages, automata (DFA, NFA, PDA, Turing machines), computability, complexity (P, NP, NP-complete), reductions |
| **Databases** | Relational model, SQL, ER modeling, normalization, query optimization, indexing, transactions (ACID), concurrency control, NoSQL intro |
| **Software Engineering** | Requirements, design patterns, testing (unit, integration), version control (Git), agile/scrum, CI/CD, software architecture |
| **Computer Networks** | OSI/TCP-IP layered model, Ethernet, IP, TCP/UDP, routing (OSPF, BGP), DNS, HTTP, sockets programming, network security basics |
| **Programming Languages** | Paradigms (imperative, functional, logic, OOP), type systems, scoping, memory management, interpreters/compilers basics, lambda calculus |

**Key Textbooks:**
- Cormen, Leiserson, Rivest & Stein (CLRS) – *Introduction to Algorithms*
- Aho, Lam, Sethi & Ullman – *Compilers: Principles, Techniques, and Tools*
- Tanenbaum & Bos – *Modern Operating Systems*
- Patterson & Hennessy – *Computer Organization and Design* (RISC-V)
- Sipser – *Introduction to the Theory of Computation*
- Kurose & Ross – *Computer Networking: A Top-Down Approach*
- Gamma, Helm, Johnson & Vlissides – *Design Patterns*
- Silberschatz, Korth & Sudarshan – *Database System Concepts*
- Abelson & Sussman – *Structure and Interpretation of Computer Programs*
- Knuth – *The Art of Computer Programming* (reference)

## 16.2 Graduate Specialization Areas

#### A. Artificial Intelligence & Machine Learning
- Machine learning (supervised, unsupervised, reinforcement), deep learning (CNNs, RNNs, Transformers), natural language processing, computer vision, generative models (GANs, diffusion, LLMs), probabilistic graphical models, multi-agent systems, AI safety, robotics/embodied AI

#### B. Systems
- Operating systems (kernel design, virtualization, containers), distributed systems (consensus, replication, consistency models — Raft, Paxos), cloud computing, storage systems, database internals, serverless, real-time systems, embedded systems

#### C. Theory of Computation
- Complexity theory (circuit complexity, communication complexity, algebraic complexity), approximation algorithms, randomized algorithms, sublinear algorithms, coding theory, quantum computing theory, cryptographic foundations, formal verification

#### D. Programming Languages & Compilers
- Type theory (dependent types, refinement types), program analysis (static, dynamic), compiler optimization (SSA, register allocation, auto-vectorization), domain-specific languages, proof assistants (Coq, Lean), language design, runtime systems, garbage collection

#### E. Computer Graphics & Visualization
- Rendering (ray tracing, path tracing, rasterization), geometry processing, physics-based simulation (fluid, cloth, rigid body), computational photography, 3D reconstruction, virtual/augmented reality, neural rendering (NeRF, Gaussian splatting)

#### F. Computer Networks & Distributed Computing
- Software-defined networking (SDN), data center networking, network verification, Internet measurement, peer-to-peer, blockchain/consensus protocols, content delivery, edge computing, 5G/6G networking

#### G. Security & Privacy
- Cryptography (post-quantum, MPC, FHE, ZKP), systems security (OS, browser, hardware), network security, malware analysis, privacy (differential privacy, federated learning), security verification, usable security

#### H. Human-Computer Interaction (HCI)
- Interaction design, usability engineering, accessibility, AR/VR interfaces, tangible computing, social computing, CSCW, computational design, crowdsourcing

#### I. Databases & Data Management
- Query processing, transaction processing, distributed databases (NewSQL, CRDTs), stream processing, graph databases, data lakes, data integration, database theory (query complexity)

#### J. Robotics & Embodied AI
- Motion planning, SLAM, manipulation, legged locomotion, multi-robot systems, sim-to-real transfer, imitation learning, robot learning from demonstration, soft robotics

- Karp (1972) — reducibility among combinatorial problems.

- Vaswani et al. (2017) — transformers.

## 16.5 Open Problems & Research Frontiers

- **P vs NP** — The most important open problem in CS/mathematics; $1M Millennium Prize problem
- **Large language models** — Scaling laws, emergent abilities, reasoning, hallucination, alignment, multimodal foundation models, efficiency (distillation, quantization, sparsity)
- **AI safety & alignment** — Ensuring AI systems behave as intended; RLHF, constitutional AI, interpretability, robustness, value alignment
- **Quantum computing** — Fault-tolerant quantum computing, quantum error correction, quantum advantage for practical problems, quantum algorithms (beyond Shor/Grover)
- **Formal verification at scale** — Verifying large software systems, verified compilers, verified OS kernels, AI-assisted proof generation
- **Privacy-preserving computation** — Practical FHE, secure multi-party computation, differential privacy at scale, private machine learning
- **Autonomous systems** — Self-driving vehicles, autonomous robots in unstructured environments, formal safety guarantees, human-robot collaboration
- **Sustainable computing** — Energy-efficient AI training/inference, green data centers, carbon-aware computing, hardware-software co-design for efficiency
- **Programming language design** — Languages that prevent bugs by construction (memory safety, concurrency safety), effect systems, gradual typing
- **Trustworthy AI** — Interpretability/explainability, fairness, robustness to distribution shift, reliable uncertainty quantification
